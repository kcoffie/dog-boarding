# Commit Message Rules

## Required
- Reference issue number in commit message (e.g., `fix: resolve login bug (#12)`)
- Use conventional commit format: `type: description`
  - `fix:` for bug fixes
  - `feat:` for new features
  - `docs:` for documentation
  - `refactor:` for refactoring
  - `test:` for tests
  - `chore:` for maintenance

## Forbidden
- Do NOT mention Claude, AI, or any AI assistant in commit messages
- Do NOT mention Claude, AI, or any AI assistant in GitHub issue comments
- Do NOT include "Generated with Claude" or similar attributions

## Examples

Good:
```
fix: resolve charlie login issue (#12)
feat: add user management to settings page (#17)
docs: update TODO list with current issues
```

Bad:
```
fix: resolve login issue (generated by Claude)
feat: add feature - Co-Authored-By: Claude
```
